## 基本ルール
- 1記事につき3-10個のタグを付与（複雑な相関関係がある場合は10個まで許容）
- 異なるカテゴリから選択
- 重要度順に記載
- 相関関係がある場合は「相関メモ」を必ず記載

## タグの拡張と管理
### 新規タグ追加の基準
1. **出現頻度**
   - 3件以上の記事で同じキーワードが重要な文脈で登場
   - 今後も継続的な言及が予想される

2. **重要性の判断**
   - 業界への影響度
   - 規制や技術的な転換点となる可能性
   - 市場規模や投資への影響

3. **カテゴリ分類**
   - 既存カテゴリへの適合性確認
   - 必要に応じて新カテゴリの作成を検討

### 追加プロセス
1. 新規タグ候補の提案
   ```markdown
   候補タグ: #新タグ名
   カテゴリ: 既存or新規
   理由: 追加が必要な理由
   関連記事: 記事タイトル1, 記事タイトル2...
   ```

2. レビューと承認
   - 提案内容の確認
   - カテゴリの適切性検証
   - 既存タグとの重複確認

3. 更新履歴の記録
   - 追加日
   - 追加理由
   - 関連する記事リンク

### タグのメンテナンス
- 四半期ごとの使用状況レビュー
- 未使用タグの整理（6ヶ月以上未使用）
- カテゴリの再編成検討

## 相関メモの記載ルール

### 相関メモの基本フォーマット
```markdown
相関メモ: [要因A] → [結果B] (時間軸: 期間, 確信度: 高/中/低)
```

### 相関メモの記載例
```markdown
相関メモ: 米国RWA規制明確化(Q4) → Ondo需要急増(遅延2-4週間, 確信度: 中)
相関メモ: ETHアップグレード発表 → L2トークン下落(即時, 確信度: 高)
相関メモ: FRB利下げ示唆 → ETHステーキング注目(1-2週間後, 確信度: 中)
```

## タグ付け例

### 基本的なタグ付け
```markdown
例：SECのステーキング規制に関する記事
#SEC #米国規制 #Staking #ETF #Ethereum
```

### 相関関係を含むタグ付け
```markdown
例：Ondo Financeのマイルストーンと規制変更の相関
#Ondo #米国規制 #Q4 #RWA #相関-正 #マクロ相関 #高影響 #中期影響
相関メモ: 米国RWA規制明確化(Q4) → Ondo機関需要増(遅延2-4週間, 確信度: 中)

例：ETHアップグレードとL2トークンの関係
#Ethereum #アップグレード #Layer2 #Polygon #Arbitrum #相関-負 #技術相関 #即時影響
相関メモ: ETHガス料金削減期待 → L2トークン売り圧力(即時, 確信度: 高)

例：マクロ経済とクリプト市場の関係
#FRB #金利政策 #Bitcoin #リスクオン #相関-正 #マクロ相関 #短期影響
相関メモ: 利下げ期待 → BTC資金流入(1-3日, 確信度: 中)
```

## 相関分析のための検索方法

### 相関関係の追跡
```markdown
# 特定プロジェクトの相関関係を調査
検索: #Ondo AND (#相関-正 OR #相関-負)

# 時期別の相関パターン調査  
検索: #Q4 AND #相関-正 AND #高影響

# ドライバー別の相関調査
検索: #マクロ相関 AND (#FRB OR #金利政策)

# 時間軸別の影響調査
検索: #即時影響 AND (#技術相関 OR #センチメント相関)
```

### 相関メモの活用
- 相関メモは記事内で「## 相関分析」セクションに記載
- 複数の相関がある場合は番号付きで整理
- 確信度が変化した場合は相関メモを更新

## 運用ルール

### 相関関係の検証
- 3回以上同様のパターンが確認された場合：確信度「高」
- 2回確認された場合：確信度「中」  
- 1回または理論的推測：確信度「低」

### タグメンテナンス
- 月次でタグ使用頻度をレビュー
- 相関メモの確信度を実績ベースで更新
- 新しい相関パターンの発見時は即座に記録

## 注意事項
- 相関関係がある記事には必ず「相関メモ」を記載
- タグは検索性と関連性を考慮して選択
- 重要度・時系列順にタグを付与
- 相関は「見せかけの相関」の可能性も考慮して慎重に判断
